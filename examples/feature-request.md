# Feature Request: ユーザーダッシュボードのリアルタイム通知機能

## 背景
現在、ユーザーダッシュボードはページリロード時にのみデータが更新される。
ユーザーからのフィードバックでは「変更があったことに気づかない」という声が多い。

## 要件

### 必須要件
1. WebSocket を使用したリアルタイム通知の実装
   - 新しいコメントが追加された時
   - Issue のステータスが変更された時
   - Pull Request がマージされた時

2. 通知バッジの表示
   - ヘッダーのベルアイコンに未読数を表示
   - クリックで通知一覧ドロップダウンを表示

3. 通知設定
   - ユーザーごとに通知の ON/OFF を設定可能
   - 通知種別ごとのフィルタリング

### 非機能要件
- WebSocket 接続は自動再接続を実装すること
- 通知の配信遅延は 3 秒以内
- 同時接続 1,000 ユーザーをサポート

## 技術的な考慮事項
- 既存の Spring Boot アプリに Spring WebSocket を追加
- 通知の永続化は PostgreSQL の notifications テーブルに保存
- Redis Pub/Sub でスケールアウト対応
